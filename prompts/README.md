# Example prompts

Copy a prompt, replace the `<…>` placeholders with your own text, and send it to
your AI assistant once the kotvmeshke MCP server is connected. Prompts are in
English (assistants follow English instructions most reliably) — the quiz
content itself can be in any language (see the last one).

Mirrors https://kotvmeshke.com/docs/mcp/examples

## A full pack from scratch

```
Create a quiz pack called "<PACK TITLE>" about <DESCRIPTION OF THE QUIZ THEME>.
Structure: 2 rounds, 3 topics per round, 5 questions per topic, easy to hard.
Questions must be specific, factual and unambiguous — exactly one correct answer each.
Where spelling may vary, add 2-3 accepted alternative answers.
Show me the outline when done.
```

## A quiz for your crowd

```
Build a fun party quiz pack about <DESCRIPTION OF THE QUIZ THEME> for <WHO WILL PLAY>.
1 round, 4 topics, 5 questions each. Keep it light: questions 1-3 easy, 4-5 tricky.
Write with humor, but keep answers factual and short (1-3 words).
Title it "<PACK TITLE>" and show me the outline.
```

## A pack from your material

```
Turn my notes below into a quiz pack called "<PACK TITLE>".
Group the material into 3-5 topics, 5 questions per topic, easy to hard.
Use only facts from my notes — do not invent anything.
Notes:
<PASTE YOUR MATERIAL>
```

## A multiple-choice quiz

```
Create a multiple-choice quiz pack called "<PACK TITLE>" about <DESCRIPTION OF THE QUIZ THEME>.
1 round, 3 topics, 5 questions each. Every question must have 4 options
with exactly one correct answer; the wrong options should be plausible.
Show me the outline when done.
```

## Add questions to a topic

```
Open my pack "<PACK TITLE>" and look at the topic "<TOPIC NAME>".
Add 5 new questions that are harder than the existing ones
and don't overlap with them. Match the style of the topic.
```

## An image on a question

```
In my pack "<PACK TITLE>", attach this image to question <QUESTION NUMBER>
of the topic "<TOPIC NAME>": <IMAGE URL>
Put it on the question itself, not on the answer reveal.
```

## Review before publishing

```
Review my pack "<PACK TITLE>" before I publish it:
1) fact-check every answer and flag anything doubtful,
2) point out ambiguous questions and suggest better wording,
3) run validation and list anything that blocks publishing.
Don't change or publish anything until I confirm.
```

## Questions in another language

```
Create a quiz pack called "<PACK TITLE>" about <DESCRIPTION OF THE QUIZ THEME>.
Write all questions and answers in <QUESTION LANGUAGE>.
2 rounds, 3 topics each, 5 questions per topic, easy to hard.
Set the pack language accordingly and show me the outline.
```
